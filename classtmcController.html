Detailed Description

Class to manage the interface to a Thorlabs Motion Controller using libftdi1

This is not thread safe. You should mutex any access to an instance of this class.

Error handling: most functions return int to indicate errors. 0 is always success, < 0 indicates an error. This is usually the error code returned by the underlying libftdi1 function.

◆ connect() [1/3]

int tmcController::connect ( bool  errmsg = true)
inline

Connect to the device.

Takes the following steps:

  1. Calls open if not already open
  2. Calls ftdi_read_chipid()
  3. Calls ftdi_set_baudrate() to set the baud rate to m_baud
  4. Calls ftdi_set_line_property() to set the serial line properties to 8-N-1
  5. Sleeps for m_preFlushSleep milliseconds
  6. Call ftdi_tcioflush()
  7. Sleeps for m_postFlushSleep milliseconds
  8. Calls ftdi_usb_reset()
  9. Calls ftdi_setflowctrl() to set SIO_RTS_CTS_HS
  10. Calls ftdi_setrts to set RTS to 1

The error code returned by each function is offset in steps of -10 to allow you to decipher which libftdi1 function failed.

Returns
0 on success
< 0 if open fails. See ftdi_usb_open_desc_index() for codes.
< -20 if ftdi_read_chipid() fails (-10 + return value)
< -30 if ftdi_set_baudrate() fails (-20 + return value)
< -40 if ftdi_set_line_property() fails (-30 + return value)
-49 if an exception is thrown durign the pre-purge sleep
< -50 if ftdi_tcioflush() fails
-59 if an exception is thrown durign the post-purge sleep
< -60 if ftdi_usb_reset() fails
< -70 if ftdi_setflowctrl() fails
< -80 if ftdi_setrts fails
Parameters
[in]errmsg[optional] flag controlling if an error message is printed on failure

◆ mod_set_chanenablestate()

int tmcController::mod_set_chanenablestate ( const uint8_t &  chnum,
const EnableState &  ces,
bool  errmsg = true 
)
inline

Enable or disable the specified drive channel.

Sends the MGMSG_MOD_SET_CHANENABLESTATE command (0x0210) See page 47 of the APT Manual

The KPZ101 sends a 10 byte reponse, after a delay, if this command changes the state. This is undocumented in the manual.
To account for this, after sending the command, this sleeps for m_postChanEnableSleep milliseconds (default 500) and then flushes the line with a read. Nothing is done with the result, and it being 0 is not an error.

Returns
0 on succcess
<0 on error from connect
<-100 on error from ftdi_write_data() (see also libusb_bulk_transfer)
<-200 on error from ftdi_read_data (see also libusb_bulk_transfer)
-666 if device not available in either ftdi_write_data() or ftdi_read_data
-700 if sleep throws an exception
-1000 if ces is EnableState::invalid
Parameters
[in]chnumthe channel number
[in]cesthe desired channel enable state, EnableState::enabled or EnableState::disabled.
[in]errmsg[optional] flag controlling if an error message is printed on failure
